# 6000C	tAIOptionTrading

An AI-driven research and backtesting pipeline for **Hang Seng Index (HSI) option trading**.

This project integrates **quantitative feature engineering**, **news retrieval and summarization**, **LLM-based market regime inference**, **scenario-to-strategy mapping**, and **option backtesting with risk controls** into one end-to-end workflow. The repository is organized around a single main pipeline entry script, `pipeline.py`, with supporting modules for features, execution, news, models, database operations, and outputs.  

## Project Objective

The goal of this project is to build a prototype system that can:

- process market and option data,
- extract structured features,
- incorporate market news as contextual input,
- infer market regimes using an LLM,
- map inferred regimes to option strategies,
- backtest the resulting decisions over time,
- and save both performance outputs and optional database records.

This repository is intended as a **research / course project prototype** rather than a production trading system.

## High-Level Workflow

The project follows a staged workflow:

1. **Load data** from the local project datasets.
2. **Build quantitative features** from index and option information.
3. **Retrieve and organize news context** for the relevant decision window.
4. **Call an LLM** to infer the market regime or scenario.
5. **Choose a corresponding option strategy** based on the inferred scenario.
6. **Run backtests** with portfolio and risk constraints.
7. **Export results** and optionally save records into a database.

## Main Entry Point

The main script is:

```bash
python pipeline.py
```

The script defines command-line arguments for backtest dates, account size, risk settings, model name, whether to search for fresh news, and whether to save results to the database.

## Command-Line Arguments

`pipeline.py` currently supports the following arguments:

| Argument | Description | Default |
|---|---|---|
| `--start` | Backtest start date | `2024-01-01` |
| `--end` | Backtest end date | `2024-12-31` |
| `--account_size` | Initial account size | `1000000` |
| `--risk_pct` | Risk percentage per trade | `0.02` |
| `--alloc_pct` | Capital allocation percentage per trade | `0.1` |
| `--stop_loss` | Stop loss rate | `0.6` |
| `--delta_limit` | Delta limit for option selection / control | `300` |
| `--model_name` | LLM model name | `deepseek-reasoner` |
| `--search_news` / `--no-search_news` | Whether to search for fresh news | `True` |
| `--save_db` / `--no-save_db` | Whether to save records into the database | `True` |

## Example Run

```bash
python pipeline.py \
  --start 2025-01-01 \
  --end 2025-12-31 \
  --risk_pct 0.05 \
  --alloc_pct 0.10 \
  --stop_loss 0.6 \
  --delta_limit 300 \
  --no-search_news \
  --no-save_db
```

## Outputs

At the end of a run, the main pipeline writes output CSV files to the `outputs/` directory:

- `monthly_summary_<run_id>.csv`
- `full_mtm_path_<run_id>.csv`

It also prints selected columns from the monthly summary and the overall metrics to the console.

## Core Modules

### `features/`
Contains quantitative feature engineering logic derived from market and option data.

### `get_news/`
Handles news search, retrieval, and source-row construction for downstream use.

### `models/`
Contains model-related logic, including keyword suggestion and trader / summarization components.

### `execution/`
Implements scenario selection, strategy mapping, backtesting, and performance summarization.

### `db/`
Provides database initialization and persistence functions for news, digestions, and performance results.

### `outputs/`
Stores exported result files generated by the pipeline.

## Environment Setup

Install dependencies with:

```bash

```

```bash
conda create -n ophr python=3.10 -y
conda activate ophr
pip install -r requirements.txt
```

> Note: the current `requirements.txt` in the repository is minimal. Depending on your local setup, additional packages may be required beyond the listed dependency.
